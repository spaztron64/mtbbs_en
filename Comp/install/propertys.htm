TMainCool, TPopupCool

property Canvas : TCanvas　（ 読取専用 ）

Canvas プロパティは，オーナー描画メニューアイテムの OnDrawItem イベントのハンドラを実現するときに使用する描画面へのアクセスを提供します。

Canvas プロパティは，メニューアイテムの項目として使用できるグラフィックイメージを描画するのに使用します。

デフォルトでは，コンポーネントがプロパティの設定値に従い，適切な描画を行ってくれます。
独自のオーナー描画をしたい場合に，OnDrawItemイベントハンドラ内でCanvasに描画してください。

例

procedure TForm1.PopupCool1DrawItem(Menu: TMenu; Item: TMenuItem;
Rect: TRect; State: TOwnerDrawState);
begin
    if odSelected in State then
    PopupCool1.Canvas.Brush.Color := clRed
  else
    PopupCool1.Canvas.Brush.Color := clBtnFace;
 
  PopupCool1.Canvas.FillRect(Rect);

  PopupCool1.Canvas.TextOut(Rect.Left + 5, Rect.Top, Item.Caption);
end;

　

　

property ActiveFont : TFont

ActiveFontプロパティは，選択中のメニューアイテムのフォントを設定するためのプロパティです。DefaultFontプロパティをTrueにした場合は，ActiveFontプロパティの設定は無効になります。

　

　

property ActiveColor : TColor

ActiveColorプロパティは，選択中のメニューアイテムの背景色を設定するためのプロパティです。

　

　

property Banner : TBanner　（TPopupCoolのみ）

Bannerプロパティは，Windows95やWindowsNT4.0の「スタート」メニューに表示されるような，縦長のタイトル等を表示する領域を提供するためのプロパティです。Bannerはグラデーション表示ができます。

TBanner = class(TPersistent)
  property ShowBanner: Boolean; // Bannerの表示・非表示
  property BannerWidth: Byte; // Bannerの幅　デフォルトは20
  property Caption: String; // Bannerに表示する文字列
  property Font: TFont; // 文字列のフォント
  property BottomColor: TColor; // グラデーション　下端の色
  property TopColor: TColor; // グラデーション　上端の色
end;

　

　

property InActiveFont : TFont

InActiveFontプロパティは，非選択状態のメニューアイテムのフォントを設定するためのプロパティです。DefaultFontプロパティをTrueにした場合は，InActiveFontプロパティの設定は無効になります。

　

　

property InActiveColor : TColor

ActiveColorプロパティは，非選択状態のメニューアイテムの背景色を設定するためのプロパティです。

　

　

property ActiveBitmap : TBitmap

ActiveBitmapプロパティは，選択中のメニューアイテムの背景のビットマップを設定するためのプロパティです。ActiveBitmapプロパティがnilでない場合は，ActioveColorプロパティの設定は無視されます。

ビットマップのサイズが，表示されるメニューの領域よりも小さい場合，ビットマップは自動的にタイル表示されます。

このプロパティに大きなサイズのビットマップを使った場合，アプリケーションのサイズも大きくなります。

　

　

property InActiveBitmap : TBitmap

InActiveBitmapプロパティは，選択中のメニューアイテムの背景のビットマップを設定するためのプロパティです。InActiveBitmapプロパティがnilでない場合は，InActioveColorプロパティの設定は無視されます。

ビットマップのサイズが，表示されるメニューの領域よりも小さい場合，ビットマップは自動的にタイル表示されます。

このプロパティに大きなサイズのビットマップを使った場合，アプリケーションのサイズも大きくなります。

　

　

property DefaultFont : Boolean

DefaultFontプロパティは，[ コントロールパネル｜画面｜デザイン]で表示されるメニューに関する設定（フォント名・サイズ）を常に使用します。このプロパティがTrueの場合，ActiveFont・InActiveFontプロパティの設定は無視されます。

　

　

property CoolMenuBar : TCoolMenuBar

Windows98のエクスプローラーライクなメニューにしたり（TCoolBarコンポーネントと共に使います），Office97アプリケーションライクなドッカブルメニューバー（Jordan Russell氏のToolbar97と共に使います）にするには，TCoolMenuBarコンポーネントを使います。CoolMenuBarプロパティにTCoolMenuBarコンポーネントを指定します。

　

property Images : TImageList

Imagesプロパティは，個々のメニューアイテムに表示するアイコンイメージのリストを提供します。アイコンファイル（32 x 32），スピードボタンに表示するビットマップファイルなどをリストの中に入れておく事ができます。メニューアイテムに表示されるイメージのサイズは，LargeImageプロパティの設定にしたがって，32 x 32か，16 x 16のどちらかになります。

ItemImages リストの各項目は，ImageIndex プロパティを設定することでイメージリストに入っているアイコンと関連付けることができます。

　

　

property ItemImages : TItemImages

// TItemImage コレクションアイテム==============================================
TItemImage = class(TCollectionItem)
public
  property Hint: String;
  property MenuItem: TMenuItem;
  property ImageIndex: Integer;
end;

// TItemImages コレクション ====================================================
TItemImages = class(TCollection)
public
  function Add: TItemImage;
  property Items[Index: Integer]: TItemImage;
end;

ItemImagesプロパティは，メニューアイテムの項目を表すItemImageオブジェクトに直接アクセスできます。

設計時には，コンポーネントをダブルクリックして表示されるコンポーネントエディターによって，各項目のHint，MenuItem，ImageIndexを簡単に設定する事が可能です。また，Delphi標準の項目エディター（オブジェクトインスペクターから起動されます）からも追加設定することが可能です。

実行時に動的に設定する場合は，addメソッドまたは，SetItemメソッドを使います。

例

var
  NewItem: TMenuItem;
  NewImages: TItemImage;
begin
  // 新しいMenuItemを動的に追加
  NewItem := TMenuItem.Create(PopupMenu1);
  NewItem.Caption := '新規メニュー';
  NewItem.OnClick := NewItemClick;
  PopupMenu1.Items.Add(NewItem);

  // メニューアイテムのイメージを指定する。addメソッド
  NewImages := PopupCool1.ItemImages.Add;
  // ImageListには別途イメージ(アイコン)を追加しておいてください
  NewImages.ImageIndex := x;
  NewImages.MenuItem := NewItem;

  // オーナードローの再実行（PopupCoolの場合は必ず実行してください。MainCoolの場合は必要ありません。）
  PopupCool1.ModifyMenuItems(PopupMenu1.Items);
end;

　

property ItemHeight : Integer

標準のメニューアイテムの高さを設定するプロパティです。デフォルト値は20です。セパレーターの高さは自動的に10ピクセルに設定されることになっています。

メニューアイテムの高さを，個別に設定したい場合は，OnMeasureItemイベントで設定することが可能です。

　

property LargeImage : Boolean

メニューアイテムに表示するアイコンイメージを32 x 32のアイコンサイズで表示したい場合は，Trueにします。16 x 16の小さいアイコンサイズの場合は，Falseにして下さい。デフォルトはFalseです。

　

　

property Cursor : TCursor

マウスカーソルがメニュー上にあるときの，カーソルの形状を設定できます。デフォルトは，crDefaultです。

メニューが表示されているときに，マウスカーソルがメニューの領域上から離れると，カーソルはcrDefaultになります。
メニュー領域上にマウスカーソルがある場合，プロパティで設定したカーソルに変わります。

　

　

property Menu : TMenu

設計時には，MainCoolコンポーネントの場合，フォーム上に配置されたMainMenuコンポーネントをリストから選択できるようになっています。PopupCoolコンポーネントの場合，フォーム上に配置されたPopupMenuコンポーネントがリストアップされているので，その中から選択できます。
動的に生成するようなとき，同一フォーム上にある（そのフォームがOWner）メニューコンポーネントを指定してください。

　

　

property MenuStyle : TMenuStyle

TMenuStyle = (msOffice, msWindows);

MenuStyleプロパティが，msOfficeの場合，Microsoft Office97アプリケーションのように，選択状態なメニューアイテムのイメージが，ボタン表示に変わります。msWindowsの場合，Windows95やWindowsNT 4.0の「スタート」メニューのように，選択状態なメニューアイテムのイメージはフルリバース（反転）表示に変わります。

　

　

property ShowHint : Boolean

マウスカーソルによって，メニューアイテムが選択状態にあるとき，ヒントウィンドウを表示するかどうかを設定するプロパティです。

Internet Exploror 4.0 のように「お気に入り」に追加したメニューアイテムのURL表示するヒントと同じことが可能になります。

表示するヒントの文字列は，ItemImagesプロパティのItemImageアイテムのHintプロパティに設定して下さい。（MenuItemのHintプロパティではありません。）コンポーネントエディター上から簡単に設定する事が可能です。複数行のヒントを表示する事もできます。

　

　

property SysMenu : TSysMenu

TSysMenu = class(TPersistent)
private
  FDrawing: Boolean;
  FItemHeight: Integer;
  FMenuStyle: TMenuStyle;

Drawingプロパティは，システムメニューをオーナードローするかどうか決めます。デフォルトはFalseです。

ItemHeightプロパティは，DrawingプロパティがTrueのとき，システムメニューのアイテムの高さを決めます。

MenuStyeプロパティは，TMainCool,TPopupCoolのMenuStyleプロパティと同じです。

（注意）上記のプロパティは，何れも設計時のみ変更が有効です。実行時に変更するとシステムメニューの表示がおかしくなります。

TCoolMenuBar

　

property Menu : TMainMenu

フォーム上のMainMenuコンポーネントを指定します。MainMenuのメニューバー領域のメニューアイテム項目がCoolMenuBarコンポーネントによって置換えられます。

　

　

property MenuCursor : TCursor

マウスカーソルがメニュー項目上にあるときの，マウスカーソルの形状を指定します。

　

　

property DefaultFont : Boolean

DefaultFontプロパティは，TMainCool,TPopupCoolのDefaultFontプロパティと同じです。

　

　

property Align : TAlign

Align プロパティは，コンテナ（親コントロール）内でコントロールを揃える方法を指定します。デフォルトはalTopです。

コンテナがTCoolBarやToolBar97の場合は，alClientに設定することをお勧めします。
